# Surfacing-Community-Leadership-Project
# {APP NAME}

A neighborhood-scoped web app that helps people turn everyday acts of helping and hosting into real relationships — and quietly surfaces the leaders already among them.

Most apps that claim to "build community" keep people on a screen. **{APP NAME} is built to do the opposite: get people off their devices and into a room together, then step back.** Success is measured by time spent with neighbors, not time spent in the app.

---

## What it does

Residents see what's happening within a few walkable blocks on an interactive map, and can:

- **Post an invitation to show up** — either an open gathering or a request for help with a task (one shared concept, one create flow).
- **Discover and RSVP** to nearby events and help requests.
- **Connect with neighbors** they've met, and invite them directly to future events — coordination only, with no open free-floating chat.
- **Follow local organizations** (schools, libraries, congregations, nonprofits) — *planned*.
- **Stay safe** with private/invite-only events, blocking, reporting, and a moderation dashboard.

---

## Philosophy

A few principles shape the product and the code:

- **A bridge, not a destination.** The app hands people off to real life and goes quiet. No endless feed, no engagement-bait.
- **Leadership, kept adjacent.** We never label anyone a "leader" — we lower the friction to act, so people build the confidence to do more.
- **Rooted in one place.** We launch one neighborhood at a time; a tool for "everywhere" feels like nowhere.
- **Coordinate, don't chat.** Messaging is always attached to an event or invite, never a general inbox.
- **Non-partisan and apolitical**, focused on neighbors helping neighbors.

## Tech stack

**Frontend**
- [Next.js](https://nextjs.org/) (React, **JavaScript**) — app UI + server-rendered public event pages for shared/QR links
- [Leaflet](https://leafletjs.com/) with OpenStreetMap tiles — the interactive map
- Tailwind CSS

**Backend**
- [FastAPI](https://fastapi.tiangolo.com/) (Python) — the API server
- [SQLAlchemy 2.0](https://www.sqlalchemy.org/) (async) + [asyncpg](https://github.com/MagicStack/asyncpg) — ORM
- [GeoAlchemy2](https://geoalchemy-2.readthedocs.io/) — PostGIS integration for "near me" queries
- [Alembic](https://alembic.sqlalchemy.org/) — database migrations
- [fastapi-users](https://fastapi-users.github.io/fastapi-users/) — auth (register, login, OAuth)
- [SQLAdmin](https://aminalaee.dev/sqladmin/) — moderation dashboard
- [Resend](https://resend.com/) — transactional email

**Database**
- [PostgreSQL](https://www.postgresql.org/) + [PostGIS](https://postgis.net/) extension

**Later (stretch features)**
- Anthropic Claude API (event-creation coach, summaries, recommendations)
- `qrcode` + WeasyPrint (flyers), `icalendar` (imported events), Redis + ARQ (background jobs)

## Getting started

### Prerequisites
- Python 3.12+
- Node.js 18+
- PostgreSQL 16+ with the PostGIS extension available
- Git

### 1. Clone the repo
```bash
git clone <repo-url>
cd {app-name}
```

### 2. Database
Create a database and enable PostGIS:
```bash
createdb leadership_db
psql leadership_db -c "CREATE EXTENSION IF NOT EXISTS postgis;"
```

### 3. Backend (FastAPI)
```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env             # then fill in the values below
alembic upgrade head             # run migrations
uvicorn app.main:app --reload    # serves at http://localhost:8000
```
Interactive API docs are auto-generated by FastAPI at `http://localhost:8000/docs`.

### 4. Frontend (Next.js)
```bash
cd frontend
npm install
cp .env.local.example .env.local # set NEXT_PUBLIC_API_URL=http://localhost:8000
npm run dev                      # serves at http://localhost:3000
```

## Environment variables

**Backend (`backend/.env`)**
```
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/stoop_dev
SECRET_KEY=change-me-to-a-long-random-string
RESEND_API_KEY=your-resend-key
FRONTEND_URL=http://localhost:3000
```

**Frontend (`frontend/.env.local`)**
```
NEXT_PUBLIC_API_URL=http://localhost:8000
```

Never commit real secrets — keep `.env` files out of version control (they're in `.gitignore`).
